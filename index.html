系统架构

四层架构，
像搭积木一样稳

🖥️
第一层：多端入口

Web 管理后台是核心操作界面；飞书作为可选的消息通道，可接收提醒、查看卡片、确认操作。

Web 后台 飞书消息 手机浏览器 主动提醒
🤖
第二层：AI 智能层

AI 助理理解自然语言，识别文件类型，多轮追问缺失信息，再调用后端 CLI 执行业务。

意图识别 多轮对话 文件识别 安全约束
⚙️
第三层：业务中台

按领域拆分的微服务模块：案件、提醒、文件、文书、人员、赔偿计算、导入等，各自独立运行。

案件管理 提醒引擎 文书生成 赔偿计算
🗄️
第四层：数据底座

业务数据存 MySQL，会话和队列用 Redis，案件文件归档到坚果云，三层分离、互不拖累。

MySQL Redis 坚果云
